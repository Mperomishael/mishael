# Firestore Database Setup Guide

This document explains how to set up your Firestore database for the portfolio management system.

## Quick Setup

### 1. Create Collections

Your Firestore database needs three collections:

#### Collection: `webDevelopment`
Stores web development projects

**Document Schema:**
```json
{
  "subcategory": "string",        // e.g., "Church Website", "E-Commerce"
  "title": "string",              // Project title
  "description": "string",        // Project description
  "imageUrl": "string",           // Firebase Storage URL
  "link": "string",               // Website URL (optional)
  "createdAt": "timestamp"        // Auto-generated by Firestore
}
```

#### Collection: `brandDesign`
Stores brand design projects

**Document Schema:**
```json
{
  "subcategory": "string",        // e.g., "Logo", "Flyers", "Church Branding"
  "title": "string",              // Design title
  "description": "string",        // Design description
  "imageUrl": "string",           // Firebase Storage URL
  "createdAt": "timestamp"        // Auto-generated by Firestore
}
```

#### Collection: `aiAutomation`
Stores AI automation projects

**Document Schema:**
```json
{
  "title": "string",              // Project title
  "description": "string",        // Project description
  "imageUrl": "string",           // Firebase Storage URL
  "createdAt": "timestamp"        // Auto-generated by Firestore
}
```

### 2. Firebase Storage Folders

Create the following folders in Firebase Storage:

- `webDevelopment/` - For web development project images
- `brandDesign/` - For brand design images
- `aiAutomation/` - For AI automation project images

These folders are automatically created when you upload the first image.

### 3. Enable Firestore Rules

In Firebase Console → Firestore Database → Rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Public read access to all portfolio collections
    match /{document=**} {
      allow read: if true;
    }
    
    // Only authenticated admins can write
    match /webDevelopment/{document=**} {
      allow write: if request.auth != null && 
                      request.auth.token.email in ['empiredigitalsworldwide@gmail.com', 'letstalk2mishael@gmail.com'];
    }
    
    match /brandDesign/{document=**} {
      allow write: if request.auth != null && 
                      request.auth.token.email in ['empiredigitalsworldwide@gmail.com', 'letstalk2mishael@gmail.com'];
    }
    
    match /aiAutomation/{document=**} {
      allow write: if request.auth != null && 
                      request.auth.token.email in ['empiredigitalsworldwide@gmail.com', 'letstalk2mishael@gmail.com'];
    }
  }
}
```

### 4. Enable Storage Rules

In Firebase Console → Storage → Rules:

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    // Public read access
    match /{allPaths=**} {
      allow read: if true;
    }
    
    // Only authenticated admins can write
    match /webDevelopment/{document=**} {
      allow write: if request.auth != null && 
                      request.auth.token.email in ['empiredigitalsworldwide@gmail.com', 'letstalk2mishael@gmail.com'];
```

## Real-time Synchronization

The portfolio sections automatically listen to Firestore in real-time:

- **Web Development Section** (`WebDevelopment.tsx`)
  - Subscribes to `webDevelopment` collection
  - Groups projects by subcategory
  - Auto-updates when admin adds/removes projects

- **Brand Design Section** (`BrandDesign.tsx`)
  - Subscribes to `brandDesign` collection
  - Groups projects by subcategory
  - Auto-updates in real-time

- **AI Automation Section** (`AIAutomation.tsx`)
  - Subscribes to `aiAutomation` collection
  - Auto-updates when projects are added/removed

## Admin Dashboard

The admin dashboard at `/admin/dashboard` allows authorized emails to:

1. **Add Projects**
   - Upload image (auto-resized to Firebase Storage)
   - Add title, description, link (for web projects)
   - Select subcategory
   - Immediate publication

2. **View Projects**
   - See thumbnails of all projects
   - Quick preview and link access

3. **Delete Projects**
   - Remove projects with confirmation
   - Image automatically deleted from Storage

## Security Notes

- ✅ Public read access for portfolio galleries
- ✅ Authentication required for admin operations
- ✅ Email validation for admin-only access
- ✅ CORS configured for image uploads
- ✅ 5MB file size limit for uploads
- ✅ Only JPG, PNG, WebP formats allowed

## Troubleshooting

### Images not showing?
1. Check Firebase Storage rules allow public read
2. Verify image URL is correct in Firestore
3. Check browser console for CORS errors

### Admin login not working?
1. Verify email is in authorized list
2. Check Firebase Auth is enabled
3. Ensure user exists in Firebase Console

### Projects not updating in real-time?
1. Check Firestore rules allow read access
2. Verify collections exist
3. Check browser console for errors
4. Reload page to manually sync
